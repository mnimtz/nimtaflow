Download & Installation

In wenigen Minuten startklar

NimtaFlow läuft per Docker auf deiner eigenen Hardware. Hol dir den Code von GitHub und starte deine Instanz.

Schnellstart (Docker)

  1. Repository holen — klone das Projekt von GitHub.
  2. .env anlegen — kopiere .env.example, setze Passwort & Foto-Pfad.
  3. Starten — docker compose up -d baut & startet alles.
  4. Quellen hinzufügen — Foto-Ordner eintragen; Indexierung läuft automatisch.
  5. iOS-App verbinden — Server-URL eintragen, anmelden, fertig.
git clone https://github.com/mnimtz/nimtaflow.git
cd nimtaflow
cp .env.example .env
docker compose up -d

Voraussetzungen

  • Docker & Docker Compose
  • Ein Verzeichnis mit deinen Fotos/Videos
  • Optional: GPU für schnellere lokale KI
  • Optional: KI-Schlüssel (Gemini/OpenAI) für Cloud-KI

Datenschutz

Datenschutz & Privatsphäre

NimtaFlow ist selbst-gehostet: deine Medien und Daten liegen auf dem von dir bzw. dem Betreiber kontrollierten Server, nicht bei einem Dritt-Dienst.

Welche Daten

  • Fotos & Videos samt Metadaten (Datum, ggf. GPS).
  • Abgeleitete Daten: Vorschaubilder, Beschreibungen/Tags, Gesichts-Merkmale, Orte.
  • Kontodaten: E-Mail, Name, verschlüsselt gespeichertes Passwort (bcrypt).

Kein Tracking, keine Werbung, keine Drittanalyse.

Verarbeitung

Standardmäßig läuft alles lokal auf dem Server. Optionale KI-Funktionen (Beschreibung, Suche, „Foto animieren") senden nur ein von dir ausgewähltes Medium an einen vom Betreiber konfigurierten KI-Anbieter — und nur, wenn aktiviert. Sind sie aus, verlässt kein Inhalt den Server.
